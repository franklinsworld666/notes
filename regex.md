正则表达式是一种处理字符串的高效解决方案。

使用正则表达式的建议：
+ 准确理解要匹配的字符串，分必须出现、可能出现、不能出现，然后把这三种组合起来
+ 完整：匹配尽可能多，避免遗漏
+ 准确：匹配尽可能少，避免匹配不必要的
+ 效率：一次编译多次使用 

## 字符组
字符组用于匹配或者不匹配某个字符。
```
[a]
[0ad]
[a-z], [0-9]
[0-9a-fA-F]
[\x00-\x7F]
[^0-9]: must match a character except for 0-9
. : match any character except for \n

# 字符组简单记法
\d: [0-9]
\w: [0-9a-zA-Z_]
\s: [ \t\r\n\v\f], 空白符
\D: [^0-9], [^\d]
\W: [^0-9a-zA-Z_]
\S: [^ \t\r\n\v\f]
[\s\S]: match any character

# POSIX 字符组，主要 linux 下的工具会用到（如 sed、awk 等）
# POSIX 字符组不能脱离[], 用法：[[:digit:]]
[:digit:]: [0-9]
[:alpha:] : [a-zA-Z]
[:alnum:] : [0-9a-zA-Z]
[:lower:]
[:upper:]
[:space:] : [ \t\r\n\v\f]
[:word:]  : [0-9a-zA-Z_]
[:xdigit:] : [0-9a-fA-F]
```

## 量词
量词，用来表示匹配的字符数量。比如，要匹配10个字母，不需要重复写10次字符组。

注意：量词逗号之后，不能有空格  
|匹配优先量词| 忽略优先量词   |   |
|---|---|---|
| {m,n}  |{m,n}? | 最少出现 m 次，最多 n 次|
| {m,}  |{m,}? | 最少出现 m 次， 隐式上限65536|
| {0,n}  | {0,n}? | 最多出现 n 次  |
| {n}  |{n}? |  出现 n 次 |
| *  | *? | 最少 0 次  |
| +  | +? | 最少 1 次  |
| ?  | ?? | 0次 或 1次  |

匹配优先量词：可以匹配的时候，优先进行匹配，会尽量匹配最长的字符串。  
忽略优先量词：可以匹配的时候，优先忽略。表现出来的效果，会尽量匹配最短的字符串。

例子：
```
# 匹配 C 语言的注释就要用忽略优先量词
/\*[\s\S]*?\*/
```

## 断言
断言，负责判断某个位置左侧或右侧的文本是否符合要求。常见的断言：单词边界、行起始/结束位置、环视。

### anchor
anchor matches a position, not the text.  

不同语言对 anchor 支持有些差异，要具体看。

+ \b 匹配这样一个位置，一边是单词字符，另一边不是单词字符。  
+ ^ head of a whole string; head of a whole string and the position after a line terminal in multiline mode
+ \A : head of a whole string in any mode  
+ $: matches the end position of a whole string or the position before a line terminal in default mode; it works in each line in multiline mode  
+ \Z : matches the end position of a whole string or the position before a line terminal in any mode  
+ \z : matches the end position of a whole string in any mode  
+

### look around
所谓环视，就是在某个位置向前或向后看，针对要匹配的条件增加一些限制条件（同时，又没有真正地去匹配, 因此理论上不影响捕获编号）。

|type   |   |
|---|---|
| (?=)   | positive look ahead   |
| (?!)   | negative look ahead  |
| (?<=)   | positive look behind  |
| (?<!)   | negative look behind  |

注意事项：
+ 环视匹配的是位置，所以可以多个环视组合实现对同一个位置的多重判断  
+ 环视组合分：嵌套，即一个包含另一个；并列  
+ look ahead 各种语言支持都很好
+ look behind 各种语言支持不一，一般要求表达式简单，同时匹配长度固定或者限制最大匹配长度。因为正则设计就是向前匹配，向后匹配是有冲突的
+ look around 中的正则表达式最好不要有括号，这会使问题复杂化，可能会有奇怪结果  
 

例子：
```
// 匹配 6 位电话号码
(?<!\d)[\d]{6}(?!\d)

// 类似 123456 加入 ，变成 123,456
// 环视嵌套
(?<=\d)(?=(\d{3})+(?!\d))

// 匹配一个数字字符串，但开头不能是 999  
(?!999)\d+\b


```

## 括号
括号可以用来分组构成子表达式，分组的目的：分支选择和引用。

### 子表达式
```
// 括号里的作为一个整体  
[1-9][\d]{14}(?:[\d]{2}[0-9x])?

// 非捕获分组
// 捕获分组，对性能影响更大；如果不需要捕获
(?:regex expression)
```

### 分支选择
```
// general forms
// notice: 应用多选结构建议不要省略括号，如果不用捕获，可以 ?:
// notice：如果 option1 and option2 开头部分重叠，会很影响效率
(option1|option2)

// 0～255
(00)?[0-9]|(0)?[0-9]{2}|1[0-9]{2}|2[0-4][0-9]|25[0-5]

// 竖线优先级较低
// begin with ab or end with cd
(^ab|cd$)
```

### 引用
分组编号根据开括号出现顺序计算，从1开始编号。编号0，默认指代整个匹配串。  

```
  (((\d{4}) - (\d{2})) - (\d{4}))
0 | ----------------------------|
1 | ----------------------------|
2  |-----------------|
3   |-----|
4             |-----|
5                        |-----| 
```

### back-reference
允许在正则表达式内部，引用之前捕获分组匹配的文本, 但不涉及位置信息。

```
// match aa, bb, cc ...
([a-z])\1

// back-reference 不涉及位置信息  
// match : cat cate
(\bcat\b)\s+\1
(\bcat\b)\s+\b\1\b  

// \10, does it mean \1 or \10
// 用 \g<1>0 mean 1， 消除二义性
// another way is 命名分组  
// 不同语言命名分组记法不同，要具体看
```

## 转义 
有些字符具有特殊含义，如果要匹配字符本身，需要进行转义。字符串到正则表达式的转义过程：  
字符串文字 ---> 正则文字 ---> 正则表达式

python 提供了 Raw String，就是可以省略字符串转义这一步，可以直接写正则文字。

```
| 字符串文字  | 正则文字  |  正则表达式|
|---|---|---|
| \\n  | \n  |  换行符|
|\\t | \t | 制表符|
|\n | 换行符| 换行符|
|\t | 制表符 | 制表符|
|\\\\| \\ | \ |
| \\" | \" | " |
| \b | 退格符 | 退格符|
|\\b | \b | 单词边界 |
| \\/| \/| /|

```

如果要特意匹配元字符，正则文字中要如下处理：
|字符   | metacharacter  |  character |
|---|---| --- |
| -  | [0-9]  | [-0-9] -紧挨[就是普通字符  |
| - | [^0-9] | [^-09] 在^后的-是普通字符|
| [ | [123] | \\[123] , [ ]都是普通字符 |
| ] } | [123] | 无法匹配时，就是普通字符 |
| ^ | [^012] |[\\^12] [0^12] ^紧跟[才是元字符|
| $ | \d$ | \d\\$|
| {  | {m,n} | \\{m,n} 匹配"{m,n}"|
| * |  | \\*|
| + |   | \\+ |
| ? |    | \\? |
| *? |  | \\*\\?|
| +? |   | \\+\\? |
| ?? |    | \\?\\? |
| .  |    | \\. |
| " | | 引号需要转义 |
| () | (ab) | \\(ab\\) 开闭括号都要转义|
| \| | (ab\|cd) | "\|"  竖线要转义|


character class 中的转义：character class 中一般把字符当成普通字符处理，只有三个需要转义（ ^ - ]）
```
// 匹配* . ?
[*.?]

// 匹配 需要转义的三个
[\^\-\]]

```



## 匹配模式
匹配时遵循的规则。常用的分四种：不区分大小写模式、单行模式、多行模式、注释模式。  

### 不区分大小写
+ 使用修饰符， (?i)
+ 使用预定义常量，不同语言名称不同

### 单行模式 --- dotall
一般情况下， `.`不能匹配`\n`，所以提供了 dotall 模式，又叫单行模式。

```
// 一般 html 跨越多行
// 或者就用 [\s\S]
(?s)<script>\s.*?</script>
```

### 多行模式
多行模式：和单行模式无关，主要影响 ^和$ 匹配规则  

字符串结尾：无行结束符，就是最后一个字符后；有行结束符，就是行结束符前。

| 符号  | 默认  | 多行模式 |
|---|---| --- |
| ^  | 字符串开头  | 字符串开头 + 每行行结束符后的位置 |
| \A  | 字符串开头  |字符串开头  |
| $  | 字符串结尾  | 字符串结尾 + 每行行结束符前的位置 |
| \Z  | 字符串结尾  | 字符串结尾 |
| \z  | 字符串最后一个字符后  | 字符串最后一个字符后 |

```
// 匹配数字结尾的行
(?m)\d$
```

### 注释模式
这块内容属于注释，对机器无效，主要给人看的。

```
// ?x  注释模式
(?mx)^   # start of a string
\d+      # digit
$        # end of a string 
```

### 表达式中的优先级
| 优先级  | 组合  |
|---|---|
| 1  | （regex)  |
| 2  | ? * +  |
| 3  | abc  |
| 4  | \|  |


### 其他
+ 不同的语言，还有其他模式，不一定有修饰符，可能是用预定义常量
+ 修饰符的作用范围，出现在括号内就对括号内起作用，出现在开头就对整个表达式起作用，出现在中间就表示从此开始起作用
+ ?- 失效修饰符，从此之后失效
+ back-reference 只会捕获内容，括号内的修饰符不会捕获  

## Unicode 编码
涉及中文字符，具体看编程语言吧。

如果正则表达式或要匹配的字符串有中文，要采用 unicode 编码。

正则表达式要采用 unicode 模式，这样字符简记法也会有不同的含义：
```
\d: 0-9 
\w: 0-9a-zA-Z_中文字符
\b: 一边满足\w , 一边不满足\w

```
