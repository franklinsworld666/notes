## 线程安全与可重入
可重入研究的是系统底层的中断函数执行顺序不同导致函数执行结果不同。
线程安全研究的是多线程执行顺序对共享资源的并发访问顺序导致的执行结果不同。


### 如何编写线程安全函数
+ 只使用局部变量，或者传入的参数；
+ 只调用线程安全的函数；
+ 使用了共享资源，但是只访问不修改；
+ 如果使用了共享资源，要进行保护；


### 如何编写可重入函数
+ 只使用局部变量，或者传入的参数；
+ 不使用全局或静态数据结构用于内部记账，如：stdio库函数，malloc；
+ 不使用经静态分配的内存来返回信息，如：strtok；
+ 不调用不可重入函数；


线程与信号：
+ 信号可以发送给进程，或者具体的线程；
+ 发送给进程的信号，进程会选择一个线程来处理；
+ 线程可以设置信号掩码，决定哪些线程可以处理进程收到的信号；
+ 对于信号的处理，是进程层面的，即：每个线程对于同一信号，处理方式是一致的；
+ 动作 stop 或 terminate，进程中的所有线程都会 stop 或 terminate；
