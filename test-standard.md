## 网络测试的目的
确定产品的功能、性能，为产品研发、改善提供支持

## 网络测试的分类
+ 协议一致性测试

测试设备对协议的实现，是否遵循了协议规范

+ 功能测试

+ 性能测试

负载测试：用于确定系统在各种工作负载下的性能，目标是测试当负载逐渐增加时，系统各项性能指标的变化情况。

压力测试：确定系统的一个瓶颈或不能接受的性能点，来获得系统能提供的最大服务级别。


## 网络层
+ 吞吐量：没有丢包的情况下，路由器能够转发的最大速率。
+ 丢包率：DUT 在不同负载和帧长度下的丢包率。在测试时，可以在吞吐量的基础上，逐渐增加负载，观察丢包情况。
+ 延迟
+ RFC1242, RFC2544, RFC2285, RFC2889

## 传输层
+ 并发 TCP 连接
+ 最大 TCP 连接建立速率
+ 最大 TCP 连接拆除速率
+ HTTP 传输速率
+ 最大 HTTP 事务处理速率
+ RFC2544, RFC2647, RFC3511 


## 网络接口层
+ 吞吐量(Throughoutput): 在没有出现丢帧的情况下，能够传输给交换机让其转发到指定输出端口的每秒最大帧数。
+ 丢帧率
+ RFC1242, RFC2544, RFC2285, RFC2889
